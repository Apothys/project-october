Case Western Reserve University

Motivation

  • As communities grow larger, it is impossible to maintain the initial feel.
  • "Initial Feel" = Thoughtful, Long-form, High Quality postings
  • Can we engineer a solution to this?
    • What is the source of the decline?

Causes of Separation From Initial Feel

  1. Users are all presented with the same content by default
    • People will tailor their submissions to their audience
  2. Customization is manual
    • Hard to find desirable sub-communities to join

Project October

Goal: Easy access to personalized content

  • Allow users to train the site to what they like, and recommend them that
  • Quickly draw inferences about an article so that it can be recommended to others

Front Page Mockup

Mining & Recommender

Essentially the approach works as follows:

  1. October predicts the user's authentic news interests regardless of the news trend, using the user's clicks in each past time period

  2. Predictions made with data in a series of past time periods are combined to gain an accurate prediction of the user's authentic news interests

  3. October predicts the user's current interests by combining their authentic news interests and the current news trend in the cluster the user belongs to within the property graph.

Mining & Recommender

Methods:

  • The text classifier will be a combination of Latent Semantic Analysis with Singular Value Decomposition and Latent Dirchlet Allocation
  • As for the learning algorithm, we will be using a custom bayesian framework.

Recommender Diagram

Architecture

  • Service Oriented Architecture (SOA)
  • Communication using Thrift Interface Definition Language
  • Frontend and Recommender both have access to Frontend database (Recommender is read-only)
  • Only Recommender interacts with graph

Other consideration: All requests are cached heavily using memcached
